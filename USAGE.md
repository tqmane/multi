# 使用例とスクリーンショットガイド

## 基本的な使い方

### ステップ1: モジュールのインストールと有効化

1. APKをビルドしてインストール
2. LSPosedマネージャーを開く
3. 「モジュール」タブで「Multi Window Patch」を有効化
4. 「スコープ」タブで「System Framework (android)」が選択されていることを確認
5. デバイスを再起動

### ステップ2: アプリを起動

任意のアプリ（例：Chrome、YouTube、Twitter など）を起動します。

### ステップ3: 通知を確認

アプリを起動すると、通知パネルに以下のような通知が表示されます：

```
🔔 マルチウィンドウ利用可能
   タップしてマルチウィンドウモードを開く
   
   [マルチウィンドウで開く]
```

### ステップ4: マルチウィンドウモードを開く

1. 通知パネルを下にスワイプ
2. 「マルチウィンドウで開く」ボタンをタップ
3. アプリが分割画面モードで起動します

## 実用例

### 例1: YouTubeとTwitterを同時に見る

1. YouTubeを起動
2. 通知から「マルチウィンドウで開く」をタップ
3. Twitterが分割画面で開きます
4. 動画を見ながらタイムラインをチェック！

### 例2: メモアプリとブラウザを並べる

1. Chromeでレシピを表示
2. 通知から「マルチウィンドウで開く」をタップ
3. メモアプリが分割画面で開きます
4. レシピを見ながらメモを取れます

### 例3: マップとメッセージアプリ

1. Google Mapsで場所を確認
2. 通知から「マルチウィンドウで開く」をタップ
3. メッセージアプリが開きます
4. 地図を見ながら待ち合わせ場所を共有！

## 通知の動作

### 通知が表示されるタイミング

- ✅ アプリを起動した時
- ✅ アプリが前面に来た時
- ✅ バックグラウンドから復帰した時

### 通知が消えるタイミング

- ✅ アプリを完全に終了した時
- ✅ タスクマネージャーからスワイプで閉じた時
- ✅ アプリが完全に終了した時

### 通知が表示されないアプリ

以下のアプリでは通知は表示されません：
- Android システム (android)
- システムUI (com.android.systemui)
- 設定 (com.android.settings)
- Multi Window Patch自体

## よくある質問

### Q: 通知がうるさくないですか？

A: いいえ、サイレント通知なので音や振動は一切ありません。通知パネルに静かに表示されるだけです。

### Q: 通知を無効化できますか？

A: はい、以下の手順で無効化できます：
1. 設定 > アプリ > Multi Window Patch > 通知
2. 「マルチウィンドウ」チャンネルをオフ

※ 通知を無効化しても、マルチウィンドウ機能は有効です。

### Q: すべてのアプリでマルチウィンドウが使えますか？

A: 技術的には可能ですが、一部のアプリ（特にゲームや銀行アプリ）では正しく動作しない場合があります。

### Q: Android 13以降で通知が表示されません

A: 通知権限を許可する必要があります：
1. 設定 > アプリ > Multi Window Patch > 権限
2. 「通知」をオンにする

### Q: 分割画面以外のマルチウィンドウモードは使えますか？

A: Android 11以降のデバイスでは、フリーフォームウィンドウモードも利用可能です（デバイスによる）。

## トラブルシューティング

### 問題: 通知が表示されない

**解決策**:
1. LSPosedでモジュールが有効か確認
2. デバイスを再起動
3. 通知権限が許可されているか確認（Android 13以降）
4. LSPosedログを確認

### 問題: ボタンを押してもマルチウィンドウが開かない

**解決策**:
1. 既に別のアプリが起動しているか確認
2. デバイスがマルチウィンドウをサポートしているか確認
3. アプリを再起動

### 問題: 特定のアプリで画面が崩れる

**解決策**:
1. そのアプリを全画面モードで使用
2. 開発者にマルチウィンドウ対応を要望
3. 必要に応じて除外リストに追加（開発者向け）

## カスタマイズ

### 特定のアプリで通知を表示しない

開発者向け：`MainHook.java`の`EXCLUDED_PACKAGES`にパッケージ名を追加してください。

詳細は `DEVELOPER.md` を参照。
